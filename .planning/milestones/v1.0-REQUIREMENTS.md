## v1 Requirements

### Proxy Core

- [x] **PRXY-01**: OpenAI-compatible `/v1/chat/completions` endpoint accepts standard OpenAI request format
- [x] **PRXY-02**: Streaming (SSE) responses with proper `text/event-stream` headers and real-time chunk delivery
- [x] **PRXY-03**: Non-streaming responses for `stream: false` requests
- [x] **PRXY-04**: `/v1/models` endpoint returns available models from configured providers
- [x] **PRXY-05**: `/health` endpoint returns proxy status for monitoring/Docker health checks
- [x] **PRXY-06**: Detailed error messages when all chain entries fail

### Chain Engine

- [x] **CHAN-01**: Configurable chains as ordered lists of provider+model pairs
- [x] **CHAN-02**: Waterfall routing -- on 429 or provider failure, automatically try next entry in chain
- [x] **CHAN-03**: Multiple named chains for different use cases

### Rate Limit Intelligence

- [x] **RATE-01**: Reactive 429 detection -- detect rate limit responses and put provider+model on cooldown
- [x] **RATE-02**: Auto-recovery -- re-enable providers when cooldown timer expires
- [x] **RATE-03**: Proactive rate limit tracking -- extract x-ratelimit-remaining/reset/limit from headers
- [x] **RATE-04**: Provider skipping -- skip providers known to be exhausted without making a request
- [x] **RATE-05**: Manual rate limit configuration per provider as fallback

### Observability

- [x] **OBSV-01**: Request logging -- provider used, model, tokens consumed, latency, HTTP status
- [x] **OBSV-02**: Per-provider usage totals -- aggregate token counts and request counts
- [x] **OBSV-03**: Per-chain usage totals -- aggregate token counts and request counts
- [x] **OBSV-04**: Live rate limit status -- remaining quota per provider

### Web UI

- [x] **WEBU-01**: Provider management -- add/remove providers with API keys
- [x] **WEBU-02**: Chain management -- create/edit/reorder provider+model pairs within chains
- [x] **WEBU-03**: Usage dashboard -- per-provider totals, per-chain totals, request log, rate limit status
- [x] **WEBU-04**: Test endpoint -- send prompt, see provider, response, and latency

### Deployment & Config

- [x] **DEPL-01**: YAML config file with Zod schema validation
- [x] **DEPL-02**: Docker deployment via docker-compose
- [x] **DEPL-03**: API key gated access
- [x] **DEPL-04**: npm/CLI package -- install globally and run via `npx 429chain`
