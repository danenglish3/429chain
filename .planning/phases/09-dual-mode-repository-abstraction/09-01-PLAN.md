**1. `src/persistence/repositories/interfaces.ts`** — Two interfaces using ONLY `import type` (zero value imports to prevent circular deps):

```typescript
import type { ProviderConfig, ChainConfig } from '../../config/types.js';
import type { ProviderUsage, ChainUsage, RequestLogRow, SummaryStats } from '../aggregator.js';
import type { RequestLogEntry } from '../request-logger.js';
```

`IAdminRepository`:
- `getConfig(): { providers: ProviderConfig[]; chains: ChainConfig[] }` — returns current provider and chain config for GET /admin/config
- `upsertProvider(provider: ProviderConfig): Promise<void>` — create or update provider in persistence (async for future Postgres)
- `deleteProvider(id: string): Promise<void>` — remove provider from persistence
- `upsertChain(chain: ChainConfig): Promise<void>` — create or update chain in persistence
- `deleteChain(name: string): Promise<void>` — remove chain from persistence

Note: `getConfig()` is sync (returns in-memory data for both SQLite and future SaaS cache). Write methods are `Promise<void>` to accommodate future async Postgres writes. The SQLite implementation will use `async` keyword to auto-wrap.

`IStatsRepository`:
- `getSummaryStats(): SummaryStats` — sync, same as UsageAggregator
- `getAllProviderUsage(): ProviderUsage[]` — sync
- `getProviderUsage(providerId: string): ProviderUsage | null` — sync
- `getAllChainUsage(): ChainUsage[]` — sync
- `getChainUsage(chainName: string): ChainUsage | null` — sync
- `getRecentRequests(limit: number): RequestLogRow[]` — sync
- `logRequest(entry: RequestLogEntry): void` — sync (fire-and-forget)

**2. `src/persistence/repositories/sqlite/admin.ts`** — `SqliteAdminRepository implements IAdminRepository`:

Constructor takes `configRef: { current: Config }` and `configPath: string`.

- `getConfig()`: returns `{ providers: configRef.current.providers, chains: configRef.current.chains }` — pure delegation, zero logic.
- `upsertProvider(provider)`: Find index in `configRef.current.providers` by `provider.id`. If found, replace at index. If not, push. Then call `writeConfig(configPath, configRef.current)`.
- `deleteProvider(id)`: Find index by id. If not found, throw `Error('Provider not found')`. Splice. Call `writeConfig(...)`.
- `upsertChain(chain)`: Find index in `configRef.current.chains` by `chain.name`. If found, replace. If not, push. Call `writeConfig(...)`.
- `deleteChain(name)`: Find index by name. If not found, throw `Error('Chain not found')`. Splice. Call `writeConfig(...)`.

Import `writeConfig` from `'../../../config/writer.js'`. Import `type Config` from `'../../../config/types.js'`.

IMPORTANT: The repository handles ONLY persistence (configRef mutation + YAML write). It does NOT handle validation, registry updates, runtime chain map updates, or default-chain guards. Those stay in the admin route handler (Plan 02).

Export:

Export `async function createRepositories(mode, deps?)`:

Parameters:
- `mode: AppMode`
- `deps?: { configRef: { current: Config }; configPath: string; aggregator: UsageAggregator; requestLogger: RequestLogger; }` — optional, required only for self-hosted

**self-hosted branch** (default):
```typescript
const { SqliteAdminRepository } = await import('./sqlite/admin.js');
const { SqliteStatsRepository } = await import('./sqlite/stats.js');
logger.info('self-hosted mode');
return {
  admin: new SqliteAdminRepository(deps!.configRef, deps!.configPath),
  stats: new SqliteStatsRepository(deps!.aggregator, deps!.requestLogger),
};
```

**saas branch:**
```typescript
const [{ SaasAdminRepository }, { SaasStatsRepository }] = await Promise.all([
  import('./saas/admin.js'),
  import('./saas/stats.js'),
]);
logger.info('saas mode');
return {
  admin: new SaasAdminRepository(),
  stats: new SaasStatsRepository(),
};
```

CRITICAL: Both branches use `await import()` (dynamic imports). This is the correct pattern for ARCH-07 — the saas files are never statically imported. Using dynamic import for self-hosted too keeps the pattern consistent and makes the factory the single point of module loading.

Note: `deps!` non-null assertion is safe because self-hosted is the default and the caller (index.ts in Plan 02) always provides deps in that mode. If someone passes `mode='self-hosted'` without deps, the runtime error is appropriate.
